\chapter*{Abstract}
\addcontentsline{toc}{chapter}{ABSTRACT}

Large language models (LLMs) offer advanced generative capabilities but are constrained by static training corpora, producing hallucinated or outdated outputs when queried about recent developments. Retrieval-Augmented Generation (RAG) mitigates this by conditioning generation on externally retrieved evidence; however, baseline vector-only retrieval often suffers from contextual imprecision.

This report presents TechPulse, a cloud-native Hybrid RAG system that synthesizes emerging technology intelligence under latency and cost constraints. The system ingests data from five heterogeneous sources---ArXiv, Hacker News, DEV.to, GitHub Trending, and curated RSS/Atom feeds---with deterministic preprocessing, token-based chunking, and semantic embedding via the MiniLM sentence-transformer. Embeddings are indexed in PostgreSQL with the pgvector extension for integrated relational filtering and vector similarity search.

Retrieval operates as a three-stage pipeline: metadata-constrained filtering, cosine similarity search, and lightweight reranking combining semantic similarity, lexical overlap, and exponential recency decay. An automatic LLM fallback chain (Groq $\to$ Bedrock $\to$ Ollama $\to$ HuggingFace) ensures generation availability. The system integrates dual-criteria drift detection, three-layer hallucination verification, and health monitoring. An evaluation framework comparing baseline and hybrid retrieval has been implemented using RAGAS metrics, grid search over reranking weights, and statistical testing (Wilcoxon signed-rank, Cohen's~$d$, bootstrap 95\% CI). This report documents the system architecture, pipeline design, implementation decisions, deviations from the original proposal, and remaining work.
