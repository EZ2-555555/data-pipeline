\chapter*{Abstract}
\addcontentsline{toc}{chapter}{ABSTRACT}

Large language models produce hallucinated or outdated outputs when queried about recent developments. Retrieval-Augmented Generation (RAG) mitigates this by conditioning generation on retrieved evidence, but baseline vector-only retrieval often suffers from contextual imprecision.

This study presents TechPulse, a cloud-native Hybrid RAG system for emerging technology intelligence under strict latency and cost constraints. The system ingests data from five heterogeneous sources (ArXiv, Hacker News, DEV.to, GitHub Trending, RSS feeds) with deterministic preprocessing, token-based chunking, and MiniLM semantic embedding, indexed in PostgreSQL with pgvector. Retrieval initially employed a weighted linear reranking formula ($\alpha$/$\beta$/$\gamma$), but design-phase sensitivity analysis revealed that optimal weights are dataset-dependent and tail latency was prohibitive (p95~=~25\,s). The system was therefore re-engineered to use parameter-free Reciprocal Rank Fusion (RRF) of semantic, BM25 lexical, and recency rankings, reducing hybrid p95 latency by 87\% to 3.36\,s while maintaining comparable retrieval quality. An automatic LLM fallback chain (Groq $\to$ Bedrock $\to$ Ollama $\to$ HuggingFace) ensures generation availability. The system integrates dual-criteria drift detection, three-layer hallucination verification, and health monitoring. Evaluation compares baseline and hybrid retrieval using RAGAS metrics and statistical testing (Wilcoxon signed-rank, Cohen's~$d$, bootstrap 95\% CI).

\noindent\textbf{Keywords:} Retrieval-Augmented Generation; Hybrid Information Retrieval; Vector Databases; Cloud-Native Architecture; MLOps; Semantic Embedding; Emerging Technology Intelligence.
